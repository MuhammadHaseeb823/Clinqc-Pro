The diagnostic logic in ClinQC Pro adheres strictly to ISO 15189:2022 requirements for Section 7.3 (Quality Control).

By combining statistical process control (SPC) with machine learning, the system provides documented traceability for:
  • Analytical sensitivity (Power of detection)
  • False rejection rates (Alert budget)
  • Method-specific remedial actions

ClinQC Pro — User Guide
A clinical-grade IQC decision support system running four parallel monitoring methods: Westgard Multirules, EWMA, Isolation Forest, and One-Class SVM. All data is stored locally in your browser — nothing leaves your device.
▶ Quick Start — 4 Steps to Your First QC Result
01
Add an Analyte
Go to Manage → Add New Analyte. Choose a preset (ALT, Glucose, etc.) or enter custom target mean (μ) and SD (σ) for each QC level (L1, L2, L3).
02
Enter QC Data
Go to Monitor → Enter QC Data. Select analyte, QC level, enter the measured value and date. All four IQC methods analyse instantly on submission.
03
Review Dashboard
Go to Monitor → Dashboard. The Command Centre shows your Levey-Jennings chart with all four method overlays, active alerts, and AI anomaly scores side by side.
04
Investigate & Report
Use Troubleshooting Guide for step-by-step root-cause investigation, then generate an Accreditation Report (PDF) for your quality management audit file.
☰ Complete Tab Reference — What Each Section Does
MONITOR
Dashboard
Command Centre
Your main monitoring screen. Shows a real-time Levey-Jennings chart with alert overlays from all four methods simultaneously. Four sub-panels display EWMA drift, Isolation Forest anomaly score, and One-Class SVM score independently so you can see where each method is alerting. The Lab Intel panel on the right gives AI-generated contextualisation for the selected analyte. Header tiles show total runs, active alerts, and overall system confidence. Click any chart to zoom.
Data needed:
At least 1 analyte configured + at least 1 QC run entered.

Outputs:
Levey-Jennings chart, alert count, method scores, Lab Intel summary.
Enter QC Data
Data Entry
Enter individual QC measurements. Select your analyte, QC level (L1/L2/L3), enter the raw measured value (in your analyte's units, e.g. mmol/L), and the date. On submission, the system automatically calculates the z-score, applies all four IQC methods, generates alerts if rules are violated, and stores the result. You can also import a CSV batch of historical QC data using the Batch Import button. Each run triggers an immediate update to the Dashboard and Alert Log.
Data needed:
Analyte configured with μ and σ targets per level.

You enter:
Measured QC value, level (L1/L2/L3), date/time.

CSV format:
date, analyte_name, level, value
QC Chart & Results
Detailed Chart View
Deep-dive chart for a selected analyte and level. Shows the full run history with Westgard rule violation flags, EWMA trend line, and AI alert markers overlaid on the z-score time series. A Method Alert Summary panel breaks down alerts per method. The Run Log table below lists every QC run with its z-score, EWMA value, and all triggered rules. A Clinical Decision Support panel provides guidance on result interpretation based on the current alert state.
Data needed:
QC runs entered for the selected analyte and level.

Outputs:
Annotated chart, run-by-run log, method breakdown, clinical guidance.
Status Grid
System Overview
A heat-map style matrix of all analytes × all QC levels showing the current status of every combination at a glance. Each cell is colour-coded: green (in-control), amber (warning), red (critical alert). Click any cell to open the Detail Inspector which shows the most recent run data, z-score, and trend chart for that analyte-level combination. Ideal for morning checks and shift handover — one screen shows the status of your entire analytical portfolio.
Data needed:
Multiple analytes with QC runs across levels.

Best used for:
Shift handover, batch release, morning overview.
MANAGE
Analyte Library
Manage Tests
View and manage all configured analytes. Each card shows the analyte name, method, unit, QC target values (μ and σ) for each level, and the total number of runs entered. You can edit analyte parameters (useful when changing reagent lots or updating target values), view performance statistics, and delete analytes. Changes to μ and σ are logged in the audit trail per ISO 15189 requirements.
Outputs:
Analyte performance cards, run counts, target values.
Add New Analyte
Setup
This is always your first step. Choose from preset common biochemistry analytes (ALT, bilirubin, creatinine, glucose, potassium, sodium, urea) or create a fully custom analyte. For each QC level (L1 low, L2 normal, L3 high) you must enter the target mean (μ) and target SD (σ) — these come from your laboratory's internal QC statistics or the QC material manufacturer's assigned values. Optionally enter a TEa% value (Total Allowable Error, from RCPAQAP or EQAS) to enable Sigma Metric calculations.
You need:
• Analyte name & units
• Method name
• μ and σ per level (from your lab's QC stats or IFU)
• TEa% (optional, for Sigma)

Where to find μ/σ:
Your lab's QC cumulative mean & SD, or QC material datasheet.

Troubleshooting Guide
Root Cause Investigation
Four tabs — one per IQC method. Each tab lists the rules/alerts that method can trigger. Click any rule (e.g. "1-3s Violation", "EWMA Drift", "Isolation Forest Outlier") to expand a structured investigation pathway including: likely analytical root causes, immediate actions to take, when to escalate, and ISO 15189-aligned remediation steps. This converts AI alerts from opaque flags into actionable clinical intelligence that lab staff can follow without needing data science expertise.
Tabs:
• Westgard Multirules
• EWMA
• Isolation Forest
• One-Class SVM

Best used when:
An alert fires and you need to know what to do next.
Benchmarking
Evidence Base
Shows the simulation study results that underpin this application — the comparative detection performance of all four methods across shift, trend, imprecision, and outlier fault scenarios. Data are from 100 independent simulation replications with IQR reported. The colour-coded table immediately shows which method detects each fault type earliest. Two charts show detection delay by scenario (bar chart) and the sensitivity vs false alarm rate trade-off for trend faults (scatter plot). This section is the evidence base justifying the multi-method hybrid monitoring strategy used throughout the app.
Outputs:
Detection delay table (IQR), bar chart, TPR vs FPR scatter.

Source:
100-replication benchmarking simulation study (manuscript in preparation).
Active Alerts
Alert Management
A live feed of all current unresolved QC alerts across all analytes, sorted by severity and recency. Each alert card shows the analyte, QC level, alert method, rule triggered, z-score value, and timestamp. Click any alert to navigate directly to the Troubleshooting Guide for that specific rule. Alerts are colour-coded: red (critical — action required before releasing results), amber (warning — investigate), blue (AI anomaly — investigate and monitor). Use this screen as your daily alert triage dashboard.
Data needed:
QC runs with triggered alerts.

Severity levels:
🔴 Critical — hold results
🟡 Warning — investigate
🔵 AI anomaly — monitor
Resolution & Audit Log
Quality Record
A chronological audit trail of all QC alerts and their resolution actions — the quality record required by ISO 15189 clause 8.7 (nonconforming work). Each entry records the analyte, date, rule violated, method, severity, and any documented corrective action. Add resolution notes to any alert entry (e.g. "Recalibration performed — cause identified as reagent lot change"). This log can be exported and included in your accreditation report. All parameter changes (μ, σ, EWMA λ, AI threshold) are also recorded here.
ISO ref:
ISO 15189:2022 cl.8.7

Outputs:
Audit trail, corrective action record, exportable log.
QC Training Lab
Educational Simulator
An interactive simulation environment for learning and teaching. Select any analyte, choose a fault type (shift, trend, imprecision, outliers) and adjust the fault magnitude using the slider. The simulator generates 100 QC runs and shows in real time how each of the four methods responds — you see the Levey-Jennings chart animate and method alerts fire at different points. A detection delay comparison panel shows how many runs each method took to detect the fault. Ideal for biomedical science students, lab induction training, and CPD. Includes a built-in quiz module.
No real data needed — runs on simulated data.

Best for:
Student training, staff CPD, understanding why AI alarms when Westgard does not.
Multivariate Drift
Cross-Analyte Analysis
Analyses whether multiple analytes are drifting together — the diagnostic signature of a systemic instrument event (reagent lot change, calibration failure, environmental shift) rather than an analyte-specific problem. Shows a Pearson correlation heatmap between all analyte z-score series (last 30 runs) and a z-score overlay chart. The alarm summary flags strong correlations (r ≥ 0.6) and concurrent directional shifts, suggesting shared causation. Use this when you see alerts on multiple analytes simultaneously and need to determine whether they share a root cause.
Data needed:
≥2 analytes with ≥3 QC runs each.

Best for:
Reagent lot change investigation, instrument troubleshooting, multi-analyte drift.
Sigma Metrics
Process Capability
Calculates the Six Sigma score for each analyte using the formula σ = (TEa − |Bias|) / CV. A sigma score ≥ 6 is world-class, ≥ 4 is acceptable, and < 3 is unacceptable requiring method review. The CV% is auto-calculated from your entered QC runs. You provide the Total Allowable Error (TEa%) from your EQA/proficiency testing scheme (e.g. RCPAQAP, UKNEQAS) and the bias from your most recent EQA result. The output includes a colour-coded gauge, grade, QC design recommendation (which Westgard rules, how many QC samples per run), and a summary table across all analytes.
You need:
• TEa% (from RCPAQAP/UKNEQAS/CLIA)
• Bias% (from EQA return)
• CV% (auto from QC data)

TEa examples:
ALT: 20% | Glucose: 10%
Na: 1.2% | K: 8.5%
Creatinine: 15%
QC Freq Advisor
Risk-Based Scheduling
Generates an ISO 15189-aligned QC frequency recommendation for each analyte based on four risk factors: clinical risk category (critical/high/moderate/low), daily patient volume, analyser stability history, and current QC runs per day. The recommendation (e.g. "Every batch", "Twice per shift", "Daily") is derived from a risk score and linked to the detection-delay data from the benchmarking study — showing exactly how many real-world hours your current QC frequency takes to detect each fault type. Includes ISO 15189:2022 clause 7.3.1 compliance context.
You enter:
• Clinical risk level
• Patient volume
• Analyser stability
• Current runs/day

Outputs:
Frequency recommendation, detection delay conversion table, ISO compliance note.

System Settings
Configuration
Configure the analytical engine. EWMA Lambda (λ): controls the exponential smoothing factor (default 0.2; lower values increase sensitivity to slow drift, higher values prioritise recent observations). AI Anomaly Sensitivity: controls the anomaly score threshold for both Isolation Forest and One-Class SVM (higher sensitivity = more alerts; lower = fewer false alarms). Use Save & Apply to persist settings — they take effect immediately on all new QC runs. Settings are stored locally and survive browser refresh. Reset to Defaults restores λ=0.2 and medium sensitivity.
Key settings:
• EWMA λ (0.05–1.0)
• AI sensitivity (1–10)
• Auto-refresh interval
• Default navigator period

Note:
Changes apply to new runs only. Historical runs retain original settings.
ⓘ The Four IQC Methods — What They Are and When Each Excels
Westgard Multirules
What it is: A hierarchy of Shewhart-based decision rules (1-3σ, 2-2σ, R-4σ, 4-1σ, 10×) applied point-by-point to each QC measurement.

Best for: Detecting sudden, large-magnitude analytical failures (reagent failure, sample mix-up, analyser malfunction). The regulatory standard — required for accreditation.

Limitation: Slow to detect gradual drift and rising imprecision. Cannot capture temporal patterns across multiple runs.
EWMA (λ=0.2)
What it is: Exponentially Weighted Moving Average — a statistical filter that accumulates evidence of small persistent shifts by weighting recent observations more heavily than older ones.

Best for: Slow calibration drift, reagent-ageing trends, systematic bias accumulation. Detects trend faults faster than Westgard (median 54 vs 57 runs).

Limitation: Smoothing suppresses intermittent outlier spikes. Poor sensitivity to rising imprecision.
Isolation Forest (AI)
What it is: An unsupervised machine learning algorithm that learns the normal multi-dimensional boundary of in-control QC behaviour and flags observations that are anomalous in that high-dimensional space.

Best for: Rising analytical imprecision (detects at median 3 runs vs 5–8 for traditional methods) and intermittent outlier patterns. Excellent variance-structure sensitivity.

Limitation: Slower than EWMA for gradual trend detection. Requires a baseline of in-control QC data to learn from.
One-Class SVM (AI)
What it is: A Support Vector Machine trained to define the minimum-volume boundary enclosing in-control QC observations in a high-dimensional kernel space. Post-training observations outside this boundary trigger alerts.

Best for: Earliest trend detection of any method tested (median 12 runs vs 54–57 for Westgard/EWMA) and very early imprecision detection (median 1 run). Highest overall sensitivity.

Limitation: Higher false alarm rate at maximum sensitivity. Calibrate AI sensitivity setting to balance alerts vs detection speed.
🔒 Data Privacy & Storage
All data is stored entirely in your browser using IndexedDB (local storage). No QC values, patient data, or laboratory information is ever transmitted to any server.

To back up your data: Use the Accreditation Report → Export CSV to save a copy of all QC runs. Keep this CSV file in your laboratory document management system.

To share the app: The entire application is a single HTML file that can be saved and shared by email or USB. Recipient data is completely separate from yours.

Clearing browser data (cache/storage) will erase all entered QC data. Export CSV regularly to prevent data loss.
💡 Tips for Clinical Use
Set up all three QC levels (L1, L2, L3) for each analyte — single-level QC misses level-specific calibration errors.

Enter TEa% in Analyte Library to unlock Sigma Metrics — this is the most important quality indicator for method performance reviews.

Use Multivariate Drift whenever you see alerts on 2+ analytes on the same day — it will tell you if they share a cause.

AI alerts (blue) do not by themselves require result withholding — they are early warning signals. Escalate to Westgard violation or clinical review before holding results.

Generate a PDF report monthly and store in your LIMS/document management system as your IQC quality record for UKAS/ISO 15189 assessment.
